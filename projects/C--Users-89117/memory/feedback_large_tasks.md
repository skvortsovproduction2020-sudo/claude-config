Перед началом любой большой задачи (крупный рефакторинг, объединение страниц, новый раздел с несколькими компонентами, миграция данных) — НЕ начинать писать код сразу.

Вместо этого написать пользователю:
> "Задача большая, предлагаю делать этапами:"
> 1. Этап 1 — ...
> 2. Этап 2 — ...
> 3. Этап 3 — ...
> "Начинаем с первого?"

Дождаться подтверждения, затем выполнять по одному этапу, каждый раз ждать "готово, продолжай".

**Why:** При попытке написать весь большой файл за один раз возникают таймауты API (Stream idle timeout), код обрывается на середине, пользователь получает нерабочий результат. Также риск ошибок выше когда пишешь большой блок без промежуточных проверок.

**How to apply:** Критерии "большой задачи": файл > 200 строк изменений, объединение 2+ страниц, новая система из нескольких API роутов + UI, любой массовый рефакторинг.
