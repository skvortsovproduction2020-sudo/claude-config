Каждый проект в Studio Cabinet имеет **собственный профиль стиля**, синтезированный из данных этого проекта.

**Правило:** если данных достаточно (голосовые источники с style_notes, TG-анализы с phrases/topics) → стиль уникальный для автора. Если данных мало или нет → универсальный экспертный fallback.

**Реализация:**
- Профиль стиля хранится в `project_data` с `key='style_profile'` (JSON-объект с полями: voice_manner, phrases, topics, what_to_avoid, structure_pattern)
- Синтезируется через `/api/style-profile/generate` — читает voice_sources, telegram_analyses, materials проекта → Claude создаёт документ
- Все генераторы (TG посты и будущие) сначала проверяют наличие `style_profile`, если есть — используют его; если нет — сырые данные из таблиц; если нет ничего → fallback

**Why:** Александр явно указал — каждый проект это отдельный автор/персонаж. Вячеслав Севрюков пишет иначе, чем другой автор. Стиль должен отражать конкретного человека.

**How to apply:** При работе с любым генератором контента — всегда проверять style_profile перед fallback. При добавлении новых генераторов — обязательно подключать к этой же системе.
