Перед любым деструктивным или необратимым действием — SQL UPDATE/DELETE, сброс данных, массовые изменения — ОБЯЗАТЕЛЬНО уточнить у пользователя:
- Какие именно записи затронуть (конкретный project_id, конкретный пользователь)
- Показать SQL/действие до выполнения и спросить подтверждение

**Why:** Однажды выполнил `UPDATE materials SET transcript = NULL WHERE type = 'instagram_reel'` без фильтра по project_id — удалил 1000 реальных Groq-транскриптов во всех проектах, не только в целевом.

**How to apply:** Любой SQL с UPDATE/DELETE/TRUNCATE — сначала показать запрос, назвать что именно будет затронуто, дождаться "да" от пользователя. Никогда не выполнять глобальные изменения по догадке о scope.
