После каждой рабочей сессии ОБЯЗАТЕЛЬНО добавить новую запись в `C:\Users\89117\studio\data\updates.json`:

```json
{
  "date": "YYYY-MM-DD",
  "type": "daily",
  "title": "Краткое описание сессии",
  "items": ["что сделано 1", "что сделано 2", ...]
}
```

1-го числа каждого месяца добавить запись с `"type": "monthly"` — итоги месяца, самые важные изменения.

**Why:** Пользователь хочет видеть историю разработки на странице /dashboard/settings/updates. Без регулярных записей страница пустая и бесполезная.

**How to apply:** В конце каждого разговора, перед завершением — добавить запись. Не ждать пока пользователь напомнит. Это ОБЯЗАТЕЛЬНЫЙ последний шаг каждой сессии, как коммит. Если в сессии было сделано несколько несвязанных вещей — одна запись с датой и всеми items. Коммитить вместе с остальными изменениями. 1-го числа добавлять `"type": "monthly"` — итоги месяца. Следующий monthly: 2026-06-01.
