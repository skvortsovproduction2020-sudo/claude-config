## ✍️ Режим Мастер Промтов

Активируется когда выбран пункт «промт» из меню, или пользователь пишет «промт», «мастер промтов», «помоги собрать промт».

**Поведение в этом режиме:**
1. Скажи: «✍️ Режим Мастер Промтов активирован. Опишите задачу — я задам уточняющие вопросы и соберём точный промт.»
2. Задавай вопросы **по одному**, не все сразу — жди ответа перед следующим
3. Вопросы: для чего промт → что на выходе → кто использует → есть пример → нужны ограничения
4. Собери финальный промт в блоке кода, готовый к копированию
5. Спроси: «Что хочешь изменить?» — дорабатывай до «ок»
6. Предложи сохранить промт в библиотеку (`promt.md`, таблица «Библиотека промтов»)

Когда пользователь говорит **«обнови handoff»**, «заканчиваем», «пауза» — обнови файл:
- `*Обновлено: YYYY-MM-DD*` — сегодняшняя дата
- `## 📍 ТЕКУЩЕЕ СОСТОЯНИЕ` — актуальный статус

### Railway деплой (khram-spas, studio-боты)
```bash
# Деплой без git push (GitHub remote может быть сломан)
railway up --detach

# Редеплой через API (если railway CLI нет)
curl -X POST https://backboard.railway.app/graphql/v2 \
  -H "Authorization: Bearer $RAILWAY_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation { deploymentRedeploy(id: \"DEPLOYMENT_ID\") { id } }"}'
```

### Supabase — создать таблицу через Management API
```bash
curl -X POST "https://api.supabase.com/v1/projects/{project_ref}/database/query" \
  -H "Authorization: Bearer $SUPABASE_PAT" \
  -H "Content-Type: application/json" \
  -d '{"query": "CREATE TABLE IF NOT EXISTS ... "}'
```

### Telegram — отправить уведомление
```js
// Базовый отправщик (parse_mode HTML для кликабельных кодов)
await fetch(`https://api.telegram.org/bot${BOT_TOKEN}/sendMessage`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    chat_id: CHAT_ID,
    text: `<b>Заголовок</b>\n\nТекст <code>КОД</code>`,
    parse_mode: 'HTML'
  })
});
```

### Max — отправить уведомление (без Bearer!)
```js
await fetch(`https://botapi.max.ru/sendMessage?chat_id=${CHAT_ID}`, {
  method: 'POST',
  headers: {
    'Authorization': MAX_BOT_TOKEN,  // без "Bearer"
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ text: 'Сообщение' })
});
```

## 📥 Режим загрузки задачи (ОБЯЗАТЕЛЬНО для всех проектов)

Активируется когда пользователь загружает задачу — голосовую расшифровку, скриншоты, описание фичи.

### ШАГ 1 — Только читать и изучать
Ничего не делать. Только читать, изучать, понимать.

### ШАГ 2 — Написать план в Telegram-формате
После изучения написать ответ СТРОГО в блоке кода (тройные обратные кавычки) — это единственный способ гарантировать, что нумерация и тире сохранятся при копировании в Telegram:

```
ЧТО Я ПОНЯЛ:
[Простым языком — что за задача, что нужно сделать]

ПЛАН:
1. [Шаг первый]
2. [Шаг второй]
3. [Шаг третий]
...

ВОПРОСЫ (если есть):
- [Уточняющий вопрос, если нужен для качественного выполнения]
```

Правила формата:
- ВЕСЬ текст плана — внутри блока кода (``` ... ```)
- Никаких markdown-таблиц
- Только нумерованные списки (1. 2. 3.) и тире (-)
- Никаких ** жирных ** и # заголовков внутри плана — только заглавные буквы для разделов
- Блок кода = копируется буква в букву, цифры и тире не теряются

### ШАГ 3 — Ждать команды пользователя

Два варианта:

1. Пользователь говорит «Погнали» или «Делай» → выполнить всё по плану, затем дать отчёт
2. Пользователь говорит «Уточнения: [текст]» → дополнить план с учётом уточнений, снова ждать

### ШАГ 4 — Отчёт после выполнения
Отчёт строго в том же формате и порядке, что и план — тоже в блоке кода:

```
ВЫПОЛНЕНО:
1. [Что сделано по шагу 1] — готово / не сработало (почему)
2. [Что сделано по шагу 2] — готово / не сработало (почему)
3. [Что сделано по шагу 3] — готово / не сработало (почему)

ИТОГ:
[Одна фраза — что в итоге работает, что нужно проверить]
```

Отчёт писать простым языком — без технических терминов, так чтобы понял человек без знания программирования.
